# Portfolio Growth Requirements — SEO Strategy
**Version:** 2.0  
**Focus:** Google Search Visibility (Organic Discovery)  
**Last Updated:** 2026-04-20

---

## Strategic Direction

**Primary Channel:** Google Search (organic)  
**Goal:** Become discoverable when target buyers search for designer + domain expertise  
**Success Definition:** High-intent leads arriving via search (freelance projects, employment offers, consulting)

---

## Audience Search Behavior (Strategic Assumptions)

### 1. Freelance Buyers / Direct Clients
**Search Intent:** Need to hire a designer for a specific project  
**Likely Keywords:**
- "AI product designer"
- "fintech UX designer"
- "data visualization designer"
- "machine learning interface designer"
- "AI product design services"
- "data-intensive UX design"

**Entry Point:** Search → Portfolio → Inquiry  

### 2. Hiring Managers / Recruiters (Employment)
**Search Intent:** Finding designers with specific expertise to hire full-time  
**Likely Keywords:**
- "AI product designer" + location / remote
- "fintech UX designer"
- "design portfolio" + AI + data
- Designer names + "portfolio"

**Entry Point:** Search → Portfolio → Conversation → Offer  

### 3. Thought Leadership / Authority Building
**Search Intent:** Learning about AI/data-intensive design, industry trends  
**Likely Keywords:**
- "designing for AI products"
- "data-intensive UX patterns"
- "machine learning interface best practices"
- Designer insights + industry knowledge

**Entry Point:** Content article → Portfolio → Deeper engagement

## Discoverability Gaps (Current State)

### Technical SEO
- ✅ Good: Semantic HTML, structured data (schema.org), meta tags, robots.txt, sitemap
- ❌ Gap: Page speed (28KB+ HTML) may impact Core Web Vitals
- ❌ Gap: Missing internal linking strategy (portfolio is single-page, limited link equity distribution)

### Content Strategy
- ❌ **Critical Gap:** No searchable content beyond portfolio page
  - Single HTML file = limited keyword targeting opportunity
  - No blog, articles, or resource pages
  - No content answering "how to design for AI," "data viz patterns," etc.
- ❌ Gap: Case studies are visual-only, not text-searchable
- ❌ Gap: No FAQ, guides, or educational content to capture long-tail keywords

### Keyword Positioning
- ❌ **Gap:** Current positioning is broad ("AI & data-intensive products")
  - Not specific enough for search optimization
  - Competing against generic "product designer" terms where big portfolio sites dominate
- ❌ Gap: No clear keyword hierarchy (primary, secondary, long-tail)

### Domain Authority
- ❌ Gap: New domain (arpitmaheshwari.com likely has low domain age / authority)
- ❌ Gap: No backlinks strategy (no guest posts, citations, partner links)
- ❌ Gap: No presence on authority sites (Medium, LinkedIn articles, industry publications)

## Requirements (Phase 2.0)

### Req 1: Content Strategy
Define a searchable content hub that ranks for target keywords while demonstrating expertise.

**Specification:**
- [ ] Identify top 20 target keywords across 3 audience segments (freelance, employment, thought leadership)
- [ ] Map keywords to search volume + intent + competition
- [ ] Create content outline: Which keywords should be answered by:
  - Portfolio page (service keywords)
  - Blog articles (educational, thought leadership)
  - Case study pages (specific project keywords)
  - Resource guides (long-tail, SEO support)
- [ ] Define minimum content pieces needed to establish ranking potential:
  - How many blog posts? What topics?
  - Case study format (text + visuals, not visuals-only)
  - Resource pages (e.g., "Data Visualization Patterns," "AI Design Principles")

**Success Metrics:**
- Page 1 ranking for 5+ primary target keywords within 6 months
- 10+ secondary keyword rankings within 9 months
- 500+ monthly organic visitors from search

### Req 2: Site Architecture Redesign
Current single-page structure limits SEO. New structure needed to support content strategy.

**Specification:**
- [ ] Define new site structure:
  - Portfolio homepage (hero + featured work)
  - Individual case study pages (ranked for specific keywords)
  - Blog / Articles section (thought leadership)
  - Resource pages (guides, patterns, educational content)
  - About page (credibility + keyword optimization)
- [ ] Map URL structure (e.g., `/case-studies/[project]`, `/articles/[topic]`, `/resources/[guide]`)
- [ ] Define internal linking strategy:
  - Case studies link to relevant articles
  - Articles link to case studies (proof + examples)
  - Resource pages link to both
- [ ] Ensure all new pages include:
  - Optimized title tags / meta descriptions (keyword inclusion)
  - Schema.org structured data (Article, Case Study, Person)
  - Open Graph / Twitter cards (social shareability)

**Success Metrics:**
- Internal linking map created (all pages interconnected)
- No orphaned pages (all pages reachable via navigation + internal links)

### Req 3: Case Study Optimization (NDA-Constrained)
Current case studies are visual-only. Need text-based versions for search + credibility, within NDA constraints.

**Constraint:** Case study business metrics are NDA-bound; can't publicly share real outcomes.

**Specification:**
- [ ] For each major case study:
  - Add searchable text describing: problem statement, design approach, methodology
  - Include *problem-context* instead of outcomes (e.g., "Redesigned ML model interface for 500K+ daily users" instead of metrics)
  - Target 1-2 secondary keywords per case study
  - Structured data (Article schema + Case Study custom schema)
- [ ] Create case study landing pages:
  - Title + keyword-optimized description
  - Problem statement + industry context (e.g., "AdTech platform struggling with model interpretability")
  - Design approach (methodology, design decisions, tradeoffs)
  - *What changed* (without metrics): "Redesigned feedback loop," "Introduced explainability layer," "Simplified decision tree"
  - Visual portfolio (existing work + before/after if possible)
  - Testimonial from client (if available, even without metrics)
  - Call-to-action (contact for similar work)
- [ ] Link case studies to relevant articles/resources

**Alternative Credibility Signals (since metrics are NDA-bound):**
- [ ] Client logos (if available) — Shows who trusted you
- [ ] Industry/sector depth (which sectors have you worked in?) — Establishes range
- [ ] Testimonials from past clients (quality of work, collaboration) — Replaces metric-based proof
- [ ] Years in space + team size — Demonstrates scale & maturity
- [ ] Speaking engagements, publications — Authority signal beyond portfolio

**Success Metrics:**
- All case studies have 500+ words of searchable, keyword-optimized text
- Each case study includes 1+ testimonial or credibility signal
- Case study pages rank for 1+ keyword each within 3 months

### Req 4: Content Hub — Substack + Guest Posts
Build thought leadership in AI/data-intensive design space. Primary vehicle: Substack.

**Rationale:** Substack drives discovery + SEO + builds owned audience (better than on-site blog for growth).

**Specification:**
- [ ] Substack Publication Setup:
  - Launch Substack newsletter ("Design for AI," "Data-Intensive Thinking," or TBD name)
  - Optimize publication title + description for search visibility
  - Cross-link to portfolio (homepage + case studies)
  - Build subscriber list as alternative lead channel
  
- [ ] Content Calendar: 1 article/month (12/year)
  - Article length: 1,500-2,500 words (balance SEO + readability)
  - Each article targets 1-2 primary keywords + 3-5 secondary keywords
  - Topics address broad "AI + data-intensive design" space (not narrowly niched)
  - Examples:
    - "Designing for Model Uncertainty: When Your Algorithm Doesn't Know"
    - "From Raw Data to User Action: Data Visualization in Product Design"
    - "Building Design Systems for ML-Powered Features"
    - "Accessible AI: Designing Interfaces Users Can Understand"
    - "The Hidden Complexity in Data-Intensive Products (And How to Simplify It)"

- [ ] Article Format:
  - Solve a real, specific problem for readers
  - Reference own case studies / work (without metrics) as proof
  - Include visuals, examples, actionable frameworks
  - Optimized title + description for SEO + Substack discoverability
  
- [ ] Distribution Strategy:
  - Publish on Substack + cross-post to LinkedIn (with Substack link)
  - Guest posts on design publications (2-3/year):
    - UX Collective, A List Apart, NN/g articles
    - fintech/adtech industry blogs (for sector-specific thought leadership)
    - Target 1 guest post per quarter minimum
  - Mention/reference in relevant design communities (Designer Hangout, Twitter, etc.)

- [ ] SEO Amplification:
  - Substack articles get indexed by Google (good SEO potential)
  - Use canonical tags if syndicating elsewhere
  - Internal linking: Case studies → relevant Substack articles
  - Portfolio homepage → links to latest article

**Success Metrics:**
- 12 articles published in Year 1 (1/month cadence met)
- 500+ Substack subscribers within 12 months
- 3-5 Substack articles ranking for target keywords (Google search)
- 2-3 guest posts published on authority sites within 12 months
- 500+ monthly organic traffic from Substack articles within 12 months
